Data Sources and Code

Each data source and the method in which the data was obtained is described below for each respective website.

NFL Team Standings

Data for the wins, losses, and ties were scraped for each team in the NFL website for tables for the years 2015 through 2019. Data for each year can be accessed by replacing the year in the URL. The information for each team was contained in one table per year but in the same location across all years. The data was then cleaned to extract only the information needed. Each year was extracted into its own CSV file and then combined into a database. The following information was updated:

  • Team names renamed across all years
  • All columns dropped except for team name, wins, losses, and ties
  • The wins, losses and ties columns were renamed to correlate with the year associated with the information
  • Pushed dictionary to MongoDB
See 2019 Standings See Data

PFR Attendance Data

Data for attendance records (total, home, and away) were scraped for tables for the years 2015 through 2019. Data for each year can be accessed by replacing the year in the URL. The information for each team was contained in one table per year but in the same location across all years. The data was then cleaned to extract only the information needed. Each year was extracted into its own CSV file and then combined into a database. The following information was updated:

  • All columns were dropped except for the team name and attendance columns (Total, Home, & Away)
  • All columns dropped except for team name, wins, losses, and ties
  • The attendance columns were then renamed so each one corresponded with the correct year
See 2019 Attendance See Data

PFR Team Data

Data for team information was scraped by replacing the team abbreviation in the URL. The information needed was saved in to a dictionary and then pushed to MongoDB. The following information was updated:

  • Team Abbreviations table built in Pandas
  • Extracted data by replacing URL team abbreviation
  • Data pushed to MongoDB
